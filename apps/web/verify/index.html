Attestation Viewer

Recover the signing address from a Sealed Inference response payload + signature. If it matches the registered TEE oracle, the response provably came from a key that was generated inside an Intel TDX + H100 enclave at provider boot.

How this works: 0G's broker signs each Sealed Inference response with a key born inside the TEE. The signed payload is keccak256(req_hash || res_hash || chatID). We ecrecover the signing address from (r, s, v) against that digest and check it matches the address registered as the oracle on the on-chain Verifier. A match means: the response provably came from this enclave.
